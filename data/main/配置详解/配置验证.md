# 配置验证和默认值处理

## 概述

配置验证系统确保 EasyDocument 的配置文件始终具有正确的数据类型和合理的默认值，同时保持向后兼容性。

## 文件结构

- `config-validator.js` - 核心验证逻辑
- `validated-config.js` - 验证后的配置模块，供其他模块导入使用

## 功能特性

### 1. 数据类型验证

系统会检查以下配置项的数据类型：
- `animation.enable` - boolean，动画总开关
- `extensions.progress_bar.enable` - boolean，进度条显示控制
- `extensions.cache_menu.enable` - boolean，缓存菜单显示控制
- `document.branch_support` - boolean，是否启用分支支持
- `document.default_branch` - string，默认分支名称
- `document.available_branches` - object（数组），可用分支列表

### 2. 默认值应用

当配置项缺失或类型错误时，系统会自动应用默认值：
- `animation.enable`: `true`
- `extensions.progress_bar.enable`: `true`
- `extensions.cache_menu.enable`: `true`
- `document.branch_support`: `false`
- `document.default_branch`: `"main"`
- `document.available_branches`: `[]`

### 3. 错误处理

- **信息日志**: 当配置项缺失时，输出信息性日志并应用默认值
- **警告日志**: 当配置项类型错误时，输出警告并使用默认值替代
- **错误日志**: 当缺少必要配置节时，输出错误信息

### 4. 向后兼容性

- 旧的配置文件无需修改即可正常工作
- 新增配置项均有合理的默认值
- 不会破坏现有功能的正常运行

### 手动验证配置

```javascript
import { validateAndApplyDefaults, getValidationReport } from './config-validator.js';

const validatedConfig = validateAndApplyDefaults(originalConfig);
const report = getValidationReport(originalConfig, validatedConfig);
```

## 验证报告

系统会在控制台输出详细的验证报告，包括：
- 应用的默认值列表
- 类型错误修正记录
- 兼容性警告信息

## 扩展验证规则

要添加新的配置验证规则，请在 `config-validator.js` 中的 `VALIDATION_RULES` 对象中添加：

```javascript
const VALIDATION_RULES = {
    'new.config.path': {
        type: 'boolean',
        default: true,
        description: '新配置项描述'
    }
};
```
