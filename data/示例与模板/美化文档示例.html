HTML 增强的美化文档示例

这个文档展示了如何使用 HTML 和 CSS 在 EasyDocument 中创建美观、交互式的文档页面，提升阅读体验。

HTML 增强的优势

虽然 Markdown 提供了简洁的语法来编写文档，但有时候我们需要更多的格式控制和交互功能。HTML 可以帮助我们实现这些高级需求。

自定义样式

使用 CSS 可以完全控制文档的外观，包括颜色、字体、间距、阴影等各个方面。

交互元素

添加按钮、标签页、折叠面板等交互元素，使文档更具交互性。

复杂布局

创建多列布局、卡片网格、响应式设计等复杂的页面结构。

提示框与警告框

在文档中，我们经常需要突出显示重要信息，可以使用不同样式的提示框。

提示： 这是一个普通提示框，用于提供额外的有用信息或建议。
警告： 这是一个警告框，提醒用户注意可能存在的风险或问题。
成功： 这是一个成功提示框，表示操作已成功完成或提供积极的反馈。

代码展示增强

为代码块添加标题和样式可以提高代码的可读性和上下文相关性。

config.js - 基本配置
const config = {
  site: {
    name: "EasyDocument",
    title: "简易静态文档系统",
    description: "一个轻量级、免编译的纯静态前端文档系统"
  },
  appearance: {
    theme_color: "#3b82f6",
    default_dark_mode: "auto"
  }
};

export default config;

步骤引导

使用自定义的步骤引导样式，可以让教程类文档更加清晰易懂。

创建项目目录

首先，创建一个新的项目目录，并将 EasyDocument 的源代码复制到该目录中。

配置网站信息

编辑 config.js 文件，设置网站的基本信息、外观和布局等。

添加文档文件

在 /data 目录中添加您的 Markdown 或 HTML 文档文件，按照逻辑结构组织目录。

生成文档结构

运行 build.py 脚本生成文档结构文件，这将自动识别您的文档并构建导航。

美化表格

标准的 HTML 表格可以通过 CSS 美化，添加颜色、阴影和悬停效果。

功能 Markdown HTML 推荐用法
基本文本格式 ✅ 简单 ✅ 可控 优先使用 Markdown
表格 ✅ 基本支持 ✅ 完全控制 复杂表格用 HTML
代码块 ✅ 语法高亮 ❌ 不支持，可自行编写 基本用 Markdown，增强用 HTML
布局控制 ❌ 不支持 ✅ 完全支持 需要布局时用 HTML
交互元素 ❌ 不支持 ✅ 支持 需要交互时用 HTML
JavaScript ❌ 不支持 ✅ 支持 加载html文件不会执行js，但是可以手动执行

结论

使用HTML和CSS可以极大地增强文档的表现力和用户体验。虽然需要更多的编码工作，但对于需要高度定制和专业外观的文档来说，这是值得的投入。

建议： 根据文档的复杂度和目标受众来选择使用Markdown还是HTML。对于简单的文档，Markdown已经足够；对于需要精细控制或特殊功能的文档，可以考虑使用HTML。
注意： 虽然支持 JavaScript，但是js代码可能不会被运行，即使点击了运行按钮，这与浏览器的安全策略有关。
